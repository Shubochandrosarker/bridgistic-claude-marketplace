# Bridgistic — Claude Code Plugin

Connect Claude Code to a WordPress site through the Bridgistic MCP server: HMAC-signed requests, scoped least-privilege keys, approvals on destructive operations, audit logs, and snapshots.

## Install

```
/plugin marketplace add Shubochandrosarker/bridgistic-claude-marketplace
/plugin install bridgistic@bridgistic-marketplace
```

## Requirements

- **Node.js 20+** on your machine (`node --version`)
- The **Bridgistic WordPress plugin** installed and activated on your site
- A **connection key** generated in **WP Admin → Bridgistic → Claude Setup**

## Configure

The plugin's MCP server reads your connection from environment variables. Set them in the shell where you run Claude Code (or in your shell profile):

```bash
export BRIDGISTIC_SITE_URL="https://example.com"
export BRIDGISTIC_KEY_ID="your_key_id"
export BRIDGISTIC_KEY_SECRET="your_key_secret"
```

For multiple sites, point `BRIDGISTIC_CONNECTIONS` at a JSON registry file instead — see [`package/connections.example.json`](package/connections.example.json).

Restart Claude Code after setting the variables, then ask Claude to run `bridgistic_get_site_info` to verify the connection.

## How the server is shipped

`server/index.js` is a self-contained, pre-built bundle of the Bridgistic MCP server (source lives in [`mcp-server/`](../../mcp-server) at the repo root). It has no runtime dependencies beyond Node.js — nothing to `npm install`. It is regenerated by `npm run build` at the repo root; do not edit it by hand.

## Safety notes

- Start with a **Read-only** key. Widen scopes only when you need them.
- Keys are scoped: the server can only do what the key's scopes allow, enforced server-side by the WordPress plugin.
- Destructive operations support dry-run, approval queues, and automatic snapshots.
- Your key secret stays in your environment — it is never sent over the wire (requests are HMAC-signed).

Troubleshooting: see [`package/TROUBLESHOOTING.md`](package/TROUBLESHOOTING.md).
